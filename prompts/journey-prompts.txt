## SESSION 1: User Journeys

```
Read CLAUDE.md and check /00-project/project-status.md for current progress.

TASK: Create user journey maps at /03-product/user-flows.md

Read these files for context:
- /00-project/concept-document.md
- /03-product/product-requirements-document.md
- /03-product/mvp-feature-list.md
- /02-strategy/value-proposition.md
- /02-strategy/positioning-strategy.md
- /02-strategy/monetization-plan.md

Agent: @product-architect

Create detailed user journey maps for 7 journeys. The first 2 are the highest 
priority — they define the core experience. Give them the most depth and detail.

For each journey, document:
- Journey name and goal
- User context (when/why they're doing this)
- Step-by-step flow (what the user does AND what the system responds, screen by screen)
- Emotional state at each step (what they're feeling)
- Key moments (the moments that make or break the experience)
- Success criteria (how we know the journey worked)
- Drop-off risks (where users might abandon, and why)
- Design implications (what this journey demands from the UX)

THE 7 JOURNEYS (in priority order):

★ JOURNEY 1: First-Time Experience (CRITICAL — highest priority, most detail)
Download → open app → onboarding → add first car → log first expense → 
see first cost insight → "aha moment"

This is the most important journey in the entire product. A user who doesn't 
complete this journey in their first session will never come back. 
Map every screen, every decision point, every possible friction.

Key questions to answer:
- How fast can we get from "just downloaded" to "I see what my car costs"?
- What's the absolute minimum info we need during onboarding?
- Should we ask for historical expenses or start fresh?
- What does the dashboard show when there's only 1 expense? 5? 10?
- When does the "aha moment" happen and how do we design for it?

★ JOURNEY 2: Daily Expense Logging (CRITICAL — the core habit loop)
Open app → quick-add expense → see updated dashboard → close app
Total time: under 30 seconds

This journey happens 2-5 times per week. If it's not fast and satisfying, 
retention dies. Every extra tap is a reason to stop using the app.

Key questions to answer:
- What's the fastest possible flow? (target: 3 taps and under 10 seconds)
- What smart defaults reduce input? (last category, today's date, suggested amounts)
- What immediate feedback makes logging feel rewarding? (updated total, streak, animation)
- What's the "hook" that makes them glance at the dashboard after logging?

JOURNEY 3: The "Aha Moment"
User has 10+ expenses over 2+ weeks → opens dashboard → sees true monthly cost 
for the first time → explores breakdown → realizes car costs more than expected → 
considers premium for deeper insights

JOURNEY 4: Vehicle Timeline
User completes a modification or repair → opens timeline → adds entry with photos 
and notes → scrolls through car's history → feels pride → shares card on social media

JOURNEY 5: Premium Upgrade
User encounters a premium feature tease → curiosity triggered → views premium 
benefits → evaluates price (€2.99/month) → decides to upgrade (or not) → 
experiences premium features for the first time

JOURNEY 6: Challenge Participation
User discovers challenges tab → sees leaderboard → joins monthly cost/km challenge → 
tracks progress → competes → sees results at month end

JOURNEY 7: Maintenance Reminder Flow
User receives reminder notification → opens app → sees what's due → 
completes maintenance → logs the expense → reminder resets

FORMAT:
For each journey, use this structure:

## Journey N: [Name]
**Priority:** ★ Critical / High / Medium
**Frequency:** [how often this happens]
**Goal:** [what the user is trying to accomplish]

### User Context
[When and why does this journey start?]

### Step-by-Step Flow
| Step | User Action | System Response | Screen | Emotional State |
|------|------------|----------------|--------|-----------------|

### Key Moments
[The 2-3 moments that make or break this journey]

### Drop-Off Risks
| Risk Point | Why They Might Leave | Mitigation |
|-----------|---------------------|------------|

### Design Implications
[What this journey demands from the UX — speed, simplicity, delight, etc.]

### Success Criteria
[How we measure if this journey works]

### Empty States
[What the user sees at each step when there's no data yet]

---

After creating the journeys, update /00-project/project-status.md.
Tell me what the recommended next tasks are.
```
